# 2026-05-09

### Decision: Pipeline shape

**What:**
```
mining → labeling → features → model → api
```

**Why:** Standard supervised-ML flow. Mining is implemented; everything else
is downstream.

Mining produces a SQLite DB at `database/codepulse.db` with five tables:

| Table           | Grain                          | Purpose                                  |
|-----------------|--------------------------------|------------------------------------------|
| `events`        | one row per (commit, file)     | the raw signal — every file touch        |
| `pull_requests` | one row per PR                 | PR metadata including labels             |
| `pr_commits`    | (pr_number, commit_hash)       | link table, parsed from commit messages  |
| `issues`        | one row per issue              | issue metadata including labels          |
| `commit_issues` | (issue_number, commit_hash)    | link table, parsed from commit messages  |

The link tables are populated by regex on commit messages
(`Merge pull request #N`, `(#N)` squash convention, `fixes #N` / `closes #N`).
A commit only links to a PR/issue if its message follows these conventions —
direct-pushed or rebased commits without these patterns will not link.

**Alternatives rejected:** none — this is the obvious shape.
